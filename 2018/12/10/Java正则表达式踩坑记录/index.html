先记录一个问题：

String#replacement(String regex, String replacement)这个方法，不会修改使用调用此方法的对象本身，而是返回一个替换后的字符串，使用debug中的calculator计算可以得到替换后的值，可是调用的text值没有改变。

思路呢，就是：

text——字符串，你手头待处理的文本

regex——正则表达式，你想要筛选的符合这个格式的内容，所以说正则表达式是一种格式

replacement——替换的目标字符串

所以，总的目的和流程就是：将text中符合regex的部分替换为replacement，至于匹配，就text.matchs(regex)

分组 ((A)(B)(C)) $1对应第一个分组A，依次类推

获取 值
m.group(0) ((A)(B)(C))
m.group(1) (A)
m.group(2) (B)
m.group(3) (C)
boolean find() 尝试查找与该模式匹配的输入序列的下一个子序列。

所以如果Mathcer匹配到了多个结果，可以使用循环取组里的值，相当于迭代器，有一个指针不停向后移动，所以text = "{name:\"Wetpaint\",created_at:{$lt:ISODate(\"${date}\"),$gt:ISODate(\"2000-01-01\")},permalink:\"abc2\"}"中m.find() 重复计算只能得到两次true

使用说明，匹配字符 使用时查阅、配合debug就可以写出正确的正则表达式了，这儿就不BB了。
